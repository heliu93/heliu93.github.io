正则表达式二

发表于 2017-01-05 | 分类于 JavaScript

在上篇博客中记录了一些正则表达式的知识，接下来继续正则表达式的探索之旅吧！

一、量词


假设场景：我们希望匹配一个连续出现20次的数字字符串，要写成\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d吗，这样就有点麻烦了，所以正则表达式用量词来解决这个问题。

使用正则表达式工具可以直观的解释上表含义，\d表示数字字符，如下：
\d? 表示出现零次或一次（最多一次）
\d+ 表示出现一次或多次（至少一次）
\d* 表示出现零次或多次（任意次）
\d{5} 表示出现5次
\d{5,10} 表示出现5到10次
\d(5,) 表示至少出现5次
\d(0,10) 表示最多10次
这样我们开头提出的问题就很好解决啦，匹配数字连续出现20次的场景，\d{20}就可以了。

看一个复杂的表达式\d{20}\w\d?\w+\d*\d{3}\w{3,5}\d{3,} 用正则表达式工具解释为：
是不是很直观呢。

二、贪婪模式与非贪婪模式


贪婪模式：
\d{3,6}可以匹配3到6次，那么正则表达式要按几次匹配呢？答案是按尽可能多的次匹配。
例：’12345678’.replace(/\d{3,6}/g,’X’)，结果为：’X78’。
非贪婪模式：让正则表达式尽可能少的匹配，也就是说一旦成功匹配不再继续尝试，就是非贪婪模式。在量词后面加上？就可以了。
例：’12345678’.replace(/\d{3,6}?/g,’X’)，结果为：’XX78’。

三、分组


例：匹配字符串Byron连续出现3次的场景，用量词可能写成Byron{3}，但实际结果并不是像预期那样，如下图：

因为量词是作用于紧挨着它的，并不能作用于整个单词，那么如何让它作用于我们想要的圈定内容呢？用分组。
使用()可以达到分组的功能，使量词作用于分组，{Byron}{3}。如下图：

例：’a1b2c3d4’.replace(/([a-z]\d){3}/g,’X’)，结果为：’Xd4’。

或


使用|可以达到或的效果，如Byron|Casper,

‘ByronCasper’.replace(/Byron|Casper/g,’X’)，结果为’XX’。
Byr(on|Ca)sper,

‘ByronsperByrCasper’.replace(/Byr(on|Ca)sper/g,’X’)，结果为’XX’。

反向引用


要求2017-01-04替换为01/04/2017
‘2017-01-04’.replace(/(\d{4})-(\d{2})-(\d{2})/g,’$2$3$1’)，结果为：01/04/2017
&代表捕获的分组内容。

忽略分组


不希望捕获某些分组，只需要在分组内加上?:就可以。
(?:Byron).(ok)，ok是第一个分组。

四、前瞻


正则表达式从文本头部向尾部开始解析，文本尾部方向，称为“前”。文本头部方向，称为“后”。
前瞻就是在正则表达式匹配到规则的时候，向前检查是否符合断言，后顾/后瞻方向相反。
JavaScript不支持后顾。
符合和不符合特定断言称为肯定/正向匹配和否定/负向匹配。

注：exp为正则部分（规则部分），assert为断言部分，比如\w(?=\d)，表示应该匹配单词字符后边有数字的情况，即满足两个条件：单词字符+数字才可以。
例：’a2*3’.replace(/\w(?=\d)/g,’X’)，结果为：’X2*3’。 只替换规则部分。

正则表达式一

发表于 2016-12-31 | 分类于 JavaScript

说起正则表达式，讲真，我第一次看到时感觉完全没办法理解，乱成一遭。于是抽时间在慕课网上自己学习关于正则表达式的知识，总结一下，与大家分享。

一、正则表达式简介


正则表达式也就是Regular Expression，是使用单个字符串来描述、匹配一系列符合某个语法规则的字符串。简单讲就是：按照某种规则去匹配符合条件的字符串。
推荐一个正则表达式工具http://regexper.com ，它会将正则表达式转换为非常直观的图形表示，可以试试。

二、RegExp对象


JavaScript通过内置对象RegExp支持正则表达式，有两种方法实例化RegExp对象：
（1）、字面量方法：
例：将is单词替换为IS
var reg = /\bis\b/;
‘He is a boy.This is a dog.Where is she?’.replace(reg,’IS’)
运行结果：
‘He IS a boy.This is a dog.Where is she?’
（注：\b为单词边界）
（2）、构造函数：
var reg = new RegExp(‘\bis\b’,’g’);
（注：\转义）

正则表达式的匹配模式支持下列3个标志，称之为修饰符：
（1）、g：global全文搜索，不添加的话，搜索到第一个匹配停止
（2）、i：ignore case忽略大小写，默认大小写敏感
（3）、m：multiple lines多行搜索

三、元字符


正则表达式由两种基本字符类型组成：原义文本字符（代表它本身含义的字符）和元字符（在正则表达式中有特殊含义的非字母字符，如：* + ？$ ^ . | \ ( ) { } [ ]）
下面列出部分元字符:
\t ：水平制表符
\v：垂直制表符
\n：换行符
\r：回车符
\0：空字符
\f：换页符
\cX：与X对应的控制字符（Ctrl+X）

四、字符类


场景：一般情况下正则表达式一个字符对应字符串的一个字符，如ab\t，假设一下，如果我们需要一类字符呢，比如要求匹配字母a或b或c呢，这个时候使用字符类就比较方便了！
使用元字符[]来构建一个简单的类；所谓类是指符合某些特性的对象，一个泛指，而不是特指某个字符；表达式[abc]把字符a或b或c归为一类，表达式可以匹配这类的字符。
例：’a1b2c3d3’.replace(/[abc]/g,’X’) 运行结果为’X1X2X3d4’

字符类取反：使用元字符^创建反向类（负向类）；反向类的意思是不属于某类的内容；表达式[^abc]表示 不是字符a或b或c的内容 例：’a1b2c3d3’.replace(/[^abc]/g,’X’) 结果为’aXbXcXXX’

五、范围类


范围类用于匹配两个字符之间的内容。可以使用[a-z]来连接两个字符表示从a到z的任意字符，这是个闭区间，也就是包含a和z本身
例：’a1b2d3x4z9’.replace(/[a-z]/g,’Q’)
结果：’Q1Q2Q3Q4Q9’
在[]组成的类内部是可以连写的，如[a-zA-Z]。
例：’a1b2d3x4z9ACD’.replace(/[a-z]/g,’Q’)
结果为：’Q1Q2Q3Q4Q9QQQ’

正则表达式提供预定义类来匹配常见的字符类，如下表：

例：匹配一个ab+数字+任意字符的字符串，可以写成： ab\d.

边界匹配字符，如下表：

例：’This is a boy’.replace(/is/g,’0’),结果为’Th0 0 a boy’
如果只想替换单词的话，这时候就需要加单词边界了，’This is a boy’.replace(/\bis\b/g,’0’),运行结果为’This 0 a boy’
（注：^放在[]内表示的是取反，如果不放在[]内则表示以XXX开始）
例：’@123@abc@’.replace(/@./g,’Q’)，结果为’Q223Qbc@’
‘@123@abc@’.replace(/^@./g,’Q’)，结果为’Q23@abc@’，只有在开头的@被替换了。$也是同样的意思。

标识符m的应用：
例：’@123
@456
@789’.replace(/^@\d/g,’X’)，运行结果为’X23
@456
@789’
如果想要匹配每一行，则需加标识符m,：’@123
@456
@789’.replace(/^@\d/g,’X’)，运行结果为’X23
X56
X89’
