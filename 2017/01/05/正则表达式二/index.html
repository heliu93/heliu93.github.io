正则表达式二

发表于 2017-01-05 | 分类于 JavaScript

在上篇博客中记录了一些正则表达式的知识，接下来继续正则表达式的探索之旅吧！

一、量词


假设场景：我们希望匹配一个连续出现20次的数字字符串，要写成\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d吗，这样就有点麻烦了，所以正则表达式用量词来解决这个问题。

使用正则表达式工具可以直观的解释上表含义，\d表示数字字符，如下：
\d? 表示出现零次或一次（最多一次）
\d+ 表示出现一次或多次（至少一次）
\d* 表示出现零次或多次（任意次）
\d{5} 表示出现5次
\d{5,10} 表示出现5到10次
\d(5,) 表示至少出现5次
\d(0,10) 表示最多10次
这样我们开头提出的问题就很好解决啦，匹配数字连续出现20次的场景，\d{20}就可以了。

二、贪婪模式与非贪婪模式


贪婪模式：
\d{3,6}可以匹配3到6次，那么正则表达式要按几次匹配呢？答案是按尽可能多的次匹配。
例：’12345678’.replace(/\d{3,6}/g,’X’)，结果为：’X78’。
非贪婪模式：让正则表达式尽可能少的匹配，也就是说一旦成功匹配不再继续尝试，就是非贪婪模式。在量词后面加上？就可以了。
例：’12345678’.replace(/\d{3,6}?/g,’X’)，结果为：’XX78’。

三、分组


例：匹配字符串Byron连续出现3次的场景，用量词可能写成Byron{3}，但实际结果并不是像预期那样，如下图：

因为量词是作用于紧挨着它的，并不能作用于整个单词，那么如何让它作用于我们想要的圈定内容呢？用分组。
使用()可以达到分组的功能，使量词作用于分组，{Byron}{3}。如下图：

例：’a1b2c3d4’.replace(/([a-z]\d){3}/g,’X’)，结果为：’Xd4’。

或


使用|可以达到或的效果，如Byron|Casper,

‘ByronCasper’.replace(/Byron|Casper/g,’X’)，结果为’XX’。
Byr(on|Ca)sper,

‘ByronsperByrCasper’.replace(/Byr(on|Ca)sper/g,’X’)，结果为’XX’。

反向引用


要求2017-01-04替换为01/04/2017
‘2017-01-04’.replace(/(\d{4})-(\d{2})-(\d{2})/g,’$2$3$1’)，结果为：01/04/2017
&代表捕获的分组内容。

忽略分组


不希望捕获某些分组，只需要在分组内加上?:就可以。
(?:Byron).(ok)，ok是第一个分组。

四、前瞻


正则表达式从文本头部向尾部开始解析，文本尾部方向，称为“前”。文本头部方向，称为“后”。
前瞻就是在正则表达式匹配到规则的时候，向前检查是否符合断言，后顾/后瞻方向相反。
JavaScript不支持后顾。
符合和不符合特定断言称为肯定/正向匹配和否定/负向匹配。

注：exp为正则部分（规则部分），assert为断言部分，比如\w(?=\d)，表示应该匹配单词字符后边有数字的情况，即满足两个条件：单词字符+数字才可以。
例：’a2*3’.replace(/\w(?=\d)/g,’X’)，结果为：’X2*3’。 只替换规则部分。
